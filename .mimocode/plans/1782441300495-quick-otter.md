# Dock 模式修复计划

## 问题分析

### 问题 1: 默认模式错误
**根因**: `app.js` 第 217 行 `state.dock = cfg.dock || true` 逻辑错误，`|| true` 导致 dock 永远为 true

**修复**: 改为 `state.dock = cfg.dock ?? false`，默认不开启 dock 模式

### 问题 2: Dock 模式不显示供应商名称
**根因**: `style.css` 第 374 行 `.card-header { display: none }` 隐藏了整个头部

**修复**: 不隐藏 card-header，改为横向内联显示徽标和供应商名

### 问题 3: 不能根据窗口大小动态变化
**根因**: 
- 没有 `@media` 查询或流式布局
- 进度条宽度固定 80px
- `flex-wrap: nowrap` 导致不换行

**修复**: 
- 添加响应式 CSS
- 进度条宽度改为百分比
- 允许换行

### 设置页面登录流程

**智谱登录**：
1. 点击"登录智谱"按钮
2. Python 打开新 PyWebView 窗口，加载 bigmodel.cn
3. 注入 JS hook 拦截 fetch/XHR
4. 用户手动登录
5. JS 检测到用量接口响应后，提取 cookie 和 usage_url
6. 关闭登录窗口，返回数据给设置页面

**OpenCode 登录**：
1. 点击"登录 OpenCode"按钮
2. Python 打开新 PyWebView 窗口，加载 opencode.ai
3. JS 监听 URL 变化，提取 workspace_id
4. 监听 cookie 变化，提取 auth cookie
5. 两者都拿到后关闭窗口
