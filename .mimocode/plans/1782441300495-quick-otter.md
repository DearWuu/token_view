# macOS 显示问题修复计划

## 问题分析

Windows 兼容性修改破坏了 macOS 显示：

### 问题 1: CSS 背景被修改
**文件**: `web/style.css` 第 42 行
- **修改前**: `background: transparent;` (macOS 需要透明)
- **修改后**: `background: var(--bg-primary);` (Windows 需要不透明)
- **影响**: macOS 透明窗口失效，显示不全

### 问题 2: moveToTop 缩小宽度
**文件**: `web/app.js` 第 258-262 行
```javascript
// 置顶后把宽度缩小 100px
setTimeout(async () => {
    const size = measurePanelSize();
    const narrowW = Math.max(260, size.width - 100);
    await window.pywebview.api.resize_window_to_content(narrowW, size.height);
}, 300);
```
- **影响**: 窗口变窄，内容显示不全

---

## 修复方案

### 修复 1: 恢复 CSS 透明背景 + 添加 Windows 类

**文件**: `web/style.css`

**修改位置**: 第 40-49 行

**修改后**:
```css
body {
    font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif;
    background: transparent;
    color: var(--text-primary);
    overflow: hidden;
    user-select: none;
    -webkit-app-region: drag;
    width: fit-content;
    height: fit-content;
}

/* Windows 非透明背景 */
body.no-transparent {
    background: var(--bg-primary);
}
```

### 修复 2: app.js 添加平台检测

**文件**: `web/app.js`

**修改位置**: init() 函数开头（第 306 行后）

**添加代码**:
```javascript
// 检测平台，Windows 上禁用透明
if (navigator.platform.indexOf('Win') !== -1 || navigator.userAgent.indexOf('Windows') !== -1) {
    document.body.classList.add('no-transparent');
}
```

### 修复 3: 移除 moveToTop 中的缩小宽度逻辑

**文件**: `web/app.js`

**删除位置**: 第 258-262 行

**删除代码**:
```javascript
// 置顶后把宽度缩小 100px（最小到 260），高度保持内容高度
setTimeout(async () => {
    const size = measurePanelSize();
    const narrowW = Math.max(260, size.width - 100);
    await window.pywebview.api.resize_window_to_content(narrowW, size.height);
}, 300);
```
